# v26.4.10 Release

This release focuses on hardening the core conversion flow and improving release/versioning consistency.

## Highlights

- Improved CurseForge client-only filtering by resolving real file names when possible, while keeping metadata lookup failures non-fatal and surfacing warnings when filtering is only partial.
- Tightened CLI validation for `--memory` and `--parallel` to prevent invalid inputs and download hangs.
- Expanded installer hash handling to support SHA-1, SHA-256, and SHA-512, and made mod hash checks case-insensitive.
- Made client-only keyword cache refresh safer by rejecting bad HTTP responses and empty keyword updates.
- Updated help text and documentation to match the current behavior, and switched the project version to `26.4.10`.
